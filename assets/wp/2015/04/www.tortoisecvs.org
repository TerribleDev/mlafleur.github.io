## What is TortoiseCVS?
<p>
TortoiseCVS lets you work with files under
<a href="http://en.wikipedia.org/wiki/Concurrent_Versions_System">CVS</a> 
version control directly from Windows Explorer. It is freely available under the
<a href="GPL.html">GPL</a>.
<p>
With TortoiseCVS you can directly check out modules, update, commit and
see differences by right clicking on files and folders within Explorer.
You can see the state of a file with overlays on top of the normal icons
within Explorer.
It even works from within the <a href="screenshot2.png">file open dialog</a>.

<p>
<img alt="TortoiseCVS at work" src="screenshot1.png"/>
<p>
You can perform tagging, branching, merging and importing, and you can go directly to a browser
web log (using <a href="http://www.lyra.org/viewcvs/">ViewCVS</a>
or <a href="http://stud.fh-heilbronn.de/~zeller/cgi/cvsweb.cgi/">CVSWeb</a>) on
a particular file.

<p>

<ul>
<li>If you want to work
with an existing repository, <a href="download.shtml">download</a>
TortoiseCVS and read the User's Guide.
<li>To create your own local repository to work alone on, <a
href="faq.shtml#cvsinit">read this FAQ entry</a>.
<li>If you want to set up a new shared repository, you will need a CVS server.  
We recommend <a href="http://www.cvsnt.org">CVSNT</a> for both Unix and Windows machines.  
TortoiseCVS can also be used with original <a href="http://ximbiot.com/cvs/">CVS</a>,
but some features depend on a <a href="http://cvsnt.org/wiki">CVSNT</a> server.
</ul>

<p>
There are some CVS things that you can't do with
TortoiseCVS - have a look at the
<a href="http://sourceforge.net/tracker/?atid=451975&group_id=48103&func=browse">feature requests database</a>.
The GPL license lets you improve TortoiseCVS, as long as you
make any changes to the source code available.  If you would like to
help implement some features,
please see the <a href="development.shtml">development</a> section.
